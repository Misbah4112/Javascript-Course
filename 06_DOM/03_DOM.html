{# //This part confuses almost everyone because there are **2 separate ideas happening**. You probably had something like: ```javascript const div = document.createElement("div") div.innerText = "Chai aur Code" const addText = document.createTextNode( "chai aur code" ) div.appendChild(addText) document.body.appendChild(div) ``` Let's execute line by line. --- ### Step 1 ```javascript const div = document.createElement("div") ``` Creates element **in memory only**. Page: ```html ``` Memory: ```html
``` Not visible yet. --- ### Step 2 ```javascript div.innerText = "Chai aur Code" ``` Add text inside div. Memory becomes: ```html
Chai aur Code
``` Still not visible. --- ### Step 3 ```javascript const addText = document.createTextNode( "chai aur code" ) ``` Creates a separate text node. Memory: ```text TextNode: "chai aur code" ``` Still invisible. --- ### Step 4 ```javascript div.appendChild(addText) ``` Attach text node inside div. Now memory becomes: ```html
Chai aur Code chai aur code
``` Notice now BOTH exist. --- ### Step 5 ```javascript document.body .appendChild(div) ``` NOW attach div to webpage. Final page: ```html
Chai aur Code chai aur code
``` Visible output: ```text Chai aur Code chai aur code ``` --- Your confusion: > why use innerText if createTextNode already exists? Answer: Usually you use **one OR the other**. Method 1: ```javascript div.innerText = "Chai aur Code" ``` OR Method 2: ```javascript const text = document.createTextNode( "Chai aur Code" ) div.appendChild(text) ``` #}
